# Feature Specification: 再利用コンポーネント集ページ

**Feature Branch**: `003-component-gallery`

**Created**: 2026-07-05

**Status**: Draft

**Input**: User description: "その他、有用なコンポーネントを収録したページを追加。スライドに差し込める再利用UIコンポーネント(コールアウト/バッジ/タグ/ボタン/データテーブル/プログレス/統計/凡例/区切り/チェックリスト/引用ブロック等)を、DSトークン準拠・WCAG AA・ハードコード禁止のままコピペで使える例と用途注記付きで収録。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - コンポーネントを見つけて差し込む (Priority: P1)

作成者が、コンポーネント集ページを見て、必要な部品(コールアウト・バッジ・データテーブル等)の見本と用途注記を確認し、そのままスライドへコピーして使える。部品はデザイントークン準拠のため、貼り付けるだけで体裁が整う。

**Why this priority**: テンプレートの実用範囲を「レイアウト」から「部品」へ広げる中核価値。単独で「準拠部品をすぐ使える」価値を提供する。

**Independent Test**: ページに 10 種以上の部品が見本 + 用途注記付きで並び、各部品がトークン由来で表示され、コントラストが AA であることを確認すれば単独で検証できる。

**Acceptance Scenarios**:

1. **Given** コンポーネント集ページ, **When** 表示する, **Then** 10 種以上の部品が見本と用途注記付きで並ぶ。
2. **Given** 任意の部品, **When** 配色・余白を確認する, **Then** すべてトークン由来で独自の色値が混在しない。
3. **Given** 状態系の部品(成功/警告/エラー), **When** 表示する, **Then** 意味づけトークン(semantic)で一貫して色分けされ、文字コントラストが AA を満たす。

---

### User Story 2 - スライドに差し込んでも崩れない (Priority: P2)

作成者が、コンポーネントを既存のスライドレイアウト内に差し込んでも、レイアウトが崩れず・準拠が保たれる(部品は自己完結したクラスで、スライドのグリッドに依存しない)。

**Why this priority**: 部品が「ギャラリーの飾り」でなく実運用で使えることを担保する増分。

**Independent Test**: 代表的な部品をスライドへ差し込んだ例が 1280×720 で崩れず表示され、トークン純度・AA を保つことを確認すれば単独で検証できる。

**Acceptance Scenarios**:

1. **Given** スライド内に差し込んだ部品, **When** 表示する, **Then** スライドが崩れず(はみ出しなし)表示される。
2. **Given** 差し込んだ部品, **When** 検証する, **Then** トークン純度(ハードコード0)と AA が保たれる。

---

### Edge Cases

- データテーブルの行数・列数が多い場合の可読性と収まり。
- コールアウトやバッジの文言が長い場合の折り返し。
- プログレス/統計の値が極端(0%/100%/大きな桁)な場合の表示。
- 状態色(警告/エラー)上の文字コントラスト。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: コンポーネント集ページを提供し、再利用可能な部品を **10 種以上**、それぞれ**見本 + 用途注記**付きで収録しなければならない(MUST)。収録候補: コールアウト(info/success/warning/error)、バッジ、タグ/チップ、ボタン(主/副/第三)、データテーブル、プログレスバー、統計(KPI)、凡例、区切り線、チェックリスト、引用ブロック、キーバリュー(定義リスト)。
- **FR-002**: すべての部品の視覚スタイルはデザイントークン由来でなければならず、独自ブランド値(ハードコード)を持ってはならない(MUST)。
- **FR-003**: 部品のテキスト/UI 要素は WCAG 2.2 AA のコントラストを満たさなければならない(MUST)。
- **FR-004**: 部品は自己完結したクラスで提供され、スライドのグリッドに依存せずスライド内へ差し込めなければならない(MUST)。
- **FR-005**: 状態系の部品(成功/警告/エラー/情報)は意味づけトークン(semantic)を一貫して用いなければならない(MUST)。
- **FR-006**: 各部品はコピーして使えるマークアップ例を伴わなければならない(MUST)。
- **FR-007**: コンポーネント集ページは、README およびショーケースから辿れなければならない(MUST)。

### Key Entities *(include if feature involves data)*

- **コンポーネント**: 再利用可能な UI 部品。名称・カテゴリ・見本・用途注記・使用トークン・コピー用マークアップを持つ。
- **コンポーネント集ページ**: コンポーネントをカテゴリ別に一覧する参照ページ。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ページに 10 種以上の部品が見本 + 用途注記付きで並ぶ。
- **SC-002**: 部品CSSの配色・余白の 100% がデザイントークン由来で、ハードコードが 0 件である。
- **SC-003**: 部品のテキスト/UI 要素の 100% が WCAG 2.2 AA を満たす。
- **SC-004**: 代表部品をスライドへ差し込んだ例が 1280×720 で崩れず表示される。
- **SC-005**: コンポーネント集ページが README とショーケースから 1 クリックで到達できる。
- **SC-006**: 各部品にコピー用マークアップ例が併記されている。

## Assumptions

- 本フィーチャーは feature 001/002 の基盤(固定トークン・セマンティック写像層・検証手段)を再利用する。
- コンポーネント集ページはスクロール型の参照ページ(16:9 スライドではない)。内容テキストは日本語を既定とする。
- 部品は静的(CSS 中心)。フォーム送信等の実挙動やアイコンフォント依存は持たない(記号/簡易 SVG で代替)。
- コピー用マークアップは各部品の近傍にコードとして併記する(専用のクリップボード機能は必須としない)。

## Out of Scope

- インタラクティブ・ウィジェット(実フォーム処理・状態管理)、JS フレームワーク。
- アイコンフォント/外部アイコンライブラリ、アニメーション。
- 編集可能 PowerPoint への変換(feature 001 と同じくフェーズ2)。
