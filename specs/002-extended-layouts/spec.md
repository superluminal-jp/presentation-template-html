# Feature Specification: 拡張スライドレイアウトセット

**Feature Branch**: `002-extended-layouts`

**Created**: 2026-07-05

**Status**: Draft

**Input**: User description: "追加のテンプレートの内容を検討 @docs/requirements.md @docs/practices.md — コア8レイアウト(feature 001)を補完する拡張レイアウトセット。ビジネス/ソフトウェア開発で頻出する表現(指標・時系列・位置づけ・手順・証言・ビジュアル・締め)を、既存の準拠制約(DSトークン・16:9・注釈・WCAG・PPT変換可能性)を継承したまま追加し、practices.md の未実演の手法を網羅的に実演する。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 頻出パターンを準拠したまま表現する (Priority: P1)

作成者が、コア8レイアウトでは扱いにくい頻出の表現(単一の重要指標、複数KPIの一覧、時系列/ロードマップ、2×2の位置づけ、手順フロー、引用/証言、全面ビジュアル、締め)を、既存テンプレートと同じ準拠スタイル(配色・タイポ・余白・注釈)で作成できる。追加レイアウトはコアと同じショーケースと単体ファイルで提供される。

**Why this priority**: 拡張セットの中核価値。頻出表現を「準拠のまま」作れることが、作成者の実務適用範囲を広げる。この 1 本で単独の価値(表現の幅)を提供できる。

**Independent Test**: 8つの拡張レイアウトに内容を入れ、16:9 で崩れず表示・PDF出力でき、配色・タイポがトークン由来であることを確認すれば単独で検証できる。

**Acceptance Scenarios**:

1. **Given** 拡張レイアウト群, **When** それぞれに内容を入力する, **Then** すべて 1280×720 で崩れず(はみ出し・見切れなし)表示される。
2. **Given** 拡張レイアウトのデック, **When** 印刷/PDF 出力する, **Then** 1 スライド = 1 ページで出力される。
3. **Given** 任意の拡張レイアウト, **When** 配色・文字色・余白を確認する, **Then** すべてブランド基準(デザイントークン)由来で、独自の色値が混在しない。
4. **Given** 拡張レイアウト, **When** 注釈レイヤーをトグルする, **Then** 既定は非表示で、切り替えで設計意図と出典が表示される。

---

### User Story 2 - ベストプラクティス網羅を完成させる (Priority: P2)

作成者/学習者が、テンプレート全体(コア + 拡張)を通じて、採用手法カタログ(practices.md)の**すべての手法**が少なくとも 1 つのスライドで実演されている状態を得られる。拡張セットは、コア8で未実演だった手法(単一指標の強調・ダッシュボード・位置づけ・社会的証明・画像優位性 等)を補って、包括的なリファレンスにする。

**Why this priority**: テンプレートを「一部の手法の例示」から「体系的な実演リファレンス」へ引き上げる。P1 の表現拡張の上に積む価値。

**Independent Test**: practices.md の全手法 ID が、コア+拡張のいずれかのスライド注釈で参照されている(未使用 0)ことを確認すれば単独で検証できる。

**Acceptance Scenarios**:

1. **Given** 拡張セット追加後の全スライド, **When** 各スライドの注釈が参照する手法 ID を集計する, **Then** practices.md の全手法が少なくとも 1 回参照される(未実演 0)。
2. **Given** 引用/証言レイアウト, **When** 社会的証明を提示する, **Then** 出典・肩書きを伴う事実として提示され、誇張・捏造の推奨文言を初期同梱しない。

---

### Edge Cases

- 単一指標(ビッグナンバー)の桁数が多い/単位が長い場合の可読性と収まり。
- 時系列/手順のステップ数が想定上限を超える場合の扱い(要素数の抑制)。
- 全面ビジュアル上に文字を重ねる場合のコントラスト確保(AA)。
- 引用文が長い場合の折り返し・収まり。
- ダッシュボードで KPI タイルが多すぎる場合の情報過多の抑制。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 拡張レイアウトセットとして 8 種を提供しなければならない(MUST): ビッグナンバー(単一指標) / KPI ダッシュボード / タイムライン(時系列・ロードマップ) / 2×2 マトリクス / プロセス(手順フロー) / 引用・証言 / 全面ビジュアル / クロージング(締め・行動喚起・連絡先)。
- **FR-002**: 各拡張レイアウトは、コア8レイアウトと同一の準拠制約を継承しなければならない(MUST): 16:9(1280×720)で崩れず表示、視覚値はデザイントークン由来のみ、注釈レイヤー(既定非表示・トグル可能)、内容保持レイアウトはアクションタイトル、WCAG 2.2 AA、印刷 1 スライド=1 ページ、PPT 変換可能な命名スロット構造。
- **FR-003**: 拡張レイアウトは、ショーケース(全体デック)および単体ファイルの双方で提供されなければならない(MUST)。
- **FR-004**: 各拡張レイアウトの注釈は、そのレイアウトが実演するベストプラクティスを採用手法カタログ(practices.md)の ID で明示しなければならない(MUST)。
- **FR-005**: 拡張セットは、コア8で未実演だった手法を補い、テンプレート全体で採用手法カタログの全手法が少なくとも 1 回実演される状態にしなければならない(MUST)。
- **FR-006**: KPI ダッシュボードレイアウトは、複数の指標タイルを提示し、任意で 1 つを「推奨/注目」として強調できなければならない(MUST)。指標値は作成者が差し込むプレースホルダとする。
- **FR-007**: 引用・証言レイアウトは、引用文に加えて出典/肩書きの提示を支えなければならない(MUST)。社会的証明は事実に基づく用途に限り、捏造・誇張の推奨文言を初期同梱してはならない(MUST)。
- **FR-008**: 全面ビジュアルレイアウトは、全面の視覚要素とキャプションを支え、重ねる文字がある場合は AA コントラストを満たさなければならない(MUST)。作成者が差し込む画像はプレースホルダとし、著作物を同梱しない(MUST)。
- **FR-009**: ビッグナンバーレイアウトは、単一の重要指標を大きく提示し、基準/比較値(アンカー)を併記でき、強調は 1 スライド 1 箇所に限らなければならない(MUST)。
- **FR-010**: タイムライン/プロセスレイアウトは、順序付きのステップと明確な順序標識(番号・矢印等)を提示しなければならない(MUST)。
- **FR-011**: 2×2 マトリクスレイアウトは、ラベル付きの 2 軸と 4 象限の内容領域を提示しなければならない(MUST)。

### Key Entities *(include if feature involves data)*

- **拡張レイアウト**: 8 種のうちの 1 つ。役割・グリッド配置・命名スロット・実演する手法 ID を持つ(コアの「スライドレイアウト」を継承)。
- **KPI タイル**: ダッシュボードの単一指標(見出し・値・補足・任意の強調フラグ)。
- **タイムライン/プロセス ステップ**: 順序・ラベル・内容を持つ順序付き要素。
- **マトリクス象限**: 2 軸上の位置(4 象限)と内容。
- **引用・出典**: 引用文と、出典/肩書き(事実ベース)。
- **ビジュアルスロット**: 全面の画像/図(作成者提供のプレースホルダ)とキャプション。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 8 つの拡張レイアウトの 100% が 16:9 で見切れなく表示され、1 スライド=1 ページで PDF 出力できる。
- **SC-002**: 拡張レイアウトの配色・タイポ・余白の 100% がデザイントークン由来で、独自ブランド値(ハードコード)が 0 件である。
- **SC-003**: 拡張レイアウトの 100% で注釈レイヤーの表示/非表示を切り替えられ、既定は非表示である。
- **SC-004**: 拡張レイアウトの主要テキストの 100% が WCAG 2.2 AA のコントラストを満たす(重ね文字を含む)。
- **SC-005**: 採用手法カタログ(practices.md)の全手法が、コア+拡張のいずれかのスライドで少なくとも 1 回実演される(未実演 0 件)。
- **SC-006**: 内容を持つ拡張レイアウトの見出しが 100% アクションタイトル形式である。
- **SC-007**: 拡張レイアウトの 100% が PPT 変換可能性検査(命名スロットの矩形確定・本文が実DOMテキスト)に合格する。
- **SC-008**: テンプレート全体のレイアウト種別数が 16(コア 8 + 拡張 8)に達し、すべてがショーケースおよび単体ファイルで提供される。
- **SC-009**: 作成者が拡張レイアウトを用いて、指標・時系列・位置づけ・証言のいずれの表現も 10 分以内に準拠状態で作成できる。

## Assumptions

- 本フィーチャーは feature 001(コア8レイアウト)を基盤とし、同一のデザイントークン(固定版)・セマンティック写像層・注釈機構・検証手段を再利用する。
- 内容テキストは日本語を既定とする。基準キャンバスは 16:9(1280×720)。
- 全面ビジュアルの画像および KPI/証言の値は作成者が差し込むプレースホルダとし、著作物・実在の推奨文言は同梱しない。
- データ表(テーブル)は当面 KPI ダッシュボード/本文レイアウトで代替し、独立レイアウトとしては本フィーチャーの必須範囲外とする(将来検討)。
- 手法の一次出典の細部は practices.md の既存記載に従う(新規手法の追加は行わず、未実演手法の実演に注力)。

## Out of Scope

- 新規ベストプラクティスの追加(本フィーチャーは既存カタログの実演完成に限定)。
- 実データのバインディングや作図エンジン、独立したデータ表レイアウト。
- アニメーション/画面遷移、編集可能 PowerPoint への実変換(feature 001 と同じくフェーズ2)。
- 多言語切替・ダークテーマ。
