# Phase 1 Data Model: コンポーネント台帳と付録/PPTX マッピング

「データ」= 実演対象コンポーネントの台帳と、その HTML 付録・PPTX への割り当て。

## エンティティ

### Component(実演対象)
- **name**: 表示名(例: コールアウト)
- **origin**: `self`(自作)| `dads`(DADS 公式)
- **states**: 提示すべき状態/バリアント(例: info/success/warning/error)
- **html_appendix**: 実演される付録スライド ID
- **pptx**: PPTX 付録での提示有無・簡略度

### Appendix Slide(付録スライド)
- **id**: 付録識別(A〜F …)
- **title**: 見出し
- **components[]**: 実演する Component の集合
- **source_markup**: 自作 CSS クラス or vendor/dads-components 由来
