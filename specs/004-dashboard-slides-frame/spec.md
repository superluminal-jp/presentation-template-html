# Feature Specification: スライド共通フレーム & ダッシュボード実践スライド

**Feature Branch**: `004-dashboard-slides-frame`

**Created**: 2026-07-05

**Status**: Draft

**Input**: User description: "各コンポーネントを使ったスライド本体のページ(付録)を追加。デジタル庁『ダッシュボードデザインの実践ガイドブックとデザインテンプレート』(https://www.digital.go.jp/resources/dashboard-guidebook)のデザインを踏襲し、デザインシステム・採用プラクティスとの整合を確認。全スライドに、左下=Copyright、右下=ページ番号、右上=取扱区分(「（組織単位）限定」「機密性1/2/3情報」)を表示。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 取扱区分と体裁の共通フレームを全スライドに付す (Priority: P1)

作成者が、官公庁・企業の配布資料として必要な取扱区分・著作権・ページ番号を、全スライドに一貫した位置で表示できる。左下に Copyright、右下にページ番号、右上に取扱区分ラベル(「（組織単位）限定」と「機密性N情報」)。区分や組織名はプレースホルダで差し替えられる。

**Why this priority**: 配布資料としての体裁・情報格付け表示は実運用の必須要件。全スライドに一貫適用される横断機能で、単独で価値を提供する。

**Independent Test**: デックの全スライドで、四隅の共通フレーム(左下 Copyright / 右下 ページ番号 / 右上 取扱区分2種)が一貫位置に表示され、機密性区分を 1/2/3 で切り替えられ、印刷時も保持されることを確認すれば単独で検証できる。

**Acceptance Scenarios**:

1. **Given** 任意のスライド, **When** 表示する, **Then** 左下に Copyright、右下にページ番号、右上に「（組織単位）限定」と「機密性N情報」が表示される。
2. **Given** デック, **When** ページ番号を確認する, **Then** 通し番号が各スライドに一意に表示される。
3. **Given** 機密性区分の設定, **When** 1/2/3 を切り替える, **Then** 右上の表示が該当区分に変わる(区分に応じた注意度の色分けを含む)。
4. **Given** 印刷/PDF 出力, **When** 出力する, **Then** 取扱区分・Copyright・ページ番号が保持される。
5. **Given** フレーム, **When** 本文と重なりを確認する, **Then** フレームは四隅の余白域に置かれ本文コンテンツと重ならない。

---

### User Story 2 - ダッシュボード実践ガイドブック準拠のスライドを付録に追加 (Priority: P2)

作成者が、デジタル庁のダッシュボードデザイン実践ガイドブックの原則(グリッド整列・KPIカード・適切なチャート選択・凡例/注釈)に沿ったダッシュボード・スライドを、付録として利用できる。多系列の配色は DS の 7 パレット(Solid Gray/Blue/Light Blue/Cyan/Green/Orange/Red)由来のカテゴリ配色を用いる。

**Why this priority**: 数値中心の意思決定資料で頻出するダッシュボード表現を、公式ガイドブック品質で提供する増分。P1 の枠組みの上で内容価値を高める。

**Independent Test**: 付録のダッシュボード・スライドが、グリッド整列の KPI カード + 多系列チャート + 凡例/注釈を含み、配色が 7 パレット由来で、1280×720 で崩れないことを確認すれば単独で検証できる。

**Acceptance Scenarios**:

1. **Given** 付録のダッシュボード・スライド, **When** 表示する, **Then** KPI カードがグリッド整列で並び、多系列チャートと凡例が伴う。
2. **Given** 多系列チャート, **When** 系列色を確認する, **Then** DS の 7 パレット由来のカテゴリ配色が用いられ、隣接系列が識別可能でコントラストが確保される。
3. **Given** ダッシュボード, **When** 採用プラクティスとの整合を確認する, **Then** チャート選択・データインク・ダッシュボード設計の指針(practices.md の C 群)に沿っている。

---

### Edge Cases

- 組織名/Copyright 文言が長い場合の折り返し・省略。
- 機密性区分が未設定の場合の既定表示(安全側の既定)。
- ページ番号: 単体スライド表示(1枚)と通しデック表示での挙動差。
- ダッシュボードの KPI カード数・系列数が多い場合の情報過多の抑制。
- 表紙・全面ビジュアル等、地色が濃いスライドでのフレーム文字のコントラスト。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 全スライドに共通フレームを表示しなければならない(MUST): 左下=Copyright、右下=ページ番号、右上=取扱区分ラベル(「（組織単位）限定」および「機密性N情報」)。
- **FR-002**: 機密性区分は 1/2/3 のいずれかを設定でき、設定に応じて右上表示が変わらなければならない(MUST)。組織単位・Copyright 文言はプレースホルダで差し替え可能でなければならない(MUST)。
- **FR-003**: ページ番号は、デック内で各スライドに一意な通し番号として表示されなければならない(MUST)。
- **FR-004**: 共通フレームは、印刷/PDF 出力時にも保持されなければならない(MUST)(注釈レイヤーとは異なり、取扱区分は配布物の要件のため常時出力)。
- **FR-005**: 共通フレームはスライド四隅の余白域に配置し、本文コンテンツと重なってはならない(MUST)。
- **FR-006**: フレームおよびダッシュボードの視覚スタイルはデザイントークン由来でなければならず、ハードコードを持ってはならない(MUST)。機密性区分は視認性のため区分別の注意色を用いてよいが、文字コントラストは WCAG 2.2 AA を満たさなければならない(MUST)。
- **FR-007**: ダッシュボード実践ガイドブックの原則(グリッド整列)に沿ったダッシュボード・スライドを付録に追加しなければならない(MUST): グリッド整列の KPI カード群 + 多系列チャート + 凡例/注釈。
- **FR-008**: 多系列チャートのカテゴリ配色は、DS の 7 パレット(Solid Gray/Blue/Light Blue/Cyan/Green/Orange/Red)由来のトークン参照で提供しなければならない(MUST)。
- **FR-009**: ダッシュボード・スライドは、採用プラクティス(practices.md の C 群: Cleveland–McGill/データインク/事前注意/Storytelling with Data/ダッシュボード設計)と整合しなければならない(MUST)。
- **FR-010**: 本フィーチャーが、ダッシュボード・ガイドブック / デザインシステム / 採用プラクティスと整合していることの確認結果を文書化しなければならない(MUST)。

### Key Entities *(include if feature involves data)*

- **スライドフレーム**: 四隅の共通表示。Copyright(左下)、ページ番号(右下)、取扱区分(右上: 組織限定 + 機密性区分)を持つ。設定値(組織名・Copyright・機密性レベル)はプレースホルダ。
- **機密性区分**: 1/2/3 のいずれか。区分名と注意色を持つ(1=公開可 / 2=要保護 / 3=厳重)。
- **ダッシュボード・スライド**: グリッド整列の KPI カード群、多系列チャート、凡例、注釈を含む付録スライド。
- **カテゴリ配色**: 多系列表示用の色集合。DS の 7 パレット由来のトークン参照。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: デックの全スライド(100%)に、四隅の共通フレーム 4 要素(Copyright/ページ番号/組織限定/機密性区分)が一貫した位置で表示される。
- **SC-002**: フレーム・ダッシュボードの配色・余白の 100% がデザイントークン由来で、ハードコードが 0 件である。
- **SC-003**: 機密性区分を 1/2/3 で切り替えると、右上表示が該当区分に変わる(3 段階すべてで確認できる)。
- **SC-004**: 付録のダッシュボード・スライドが、グリッド整列の KPI カード + 多系列チャート + 凡例を含み、1280×720 で崩れず表示・印刷できる。
- **SC-005**: 多系列カテゴリ配色が DS の 7 パレット由来で、主要テキスト/系列ラベルのコントラストが WCAG 2.2 AA を満たす。
- **SC-006**: 共通フレーム(取扱区分・Copyright・ページ番号)が印刷/PDF 出力でも保持される。
- **SC-007**: ガイドブック / DS / 採用プラクティスとの整合確認が文書に記録される。

## Assumptions

- 本フィーチャーは feature 001–003 の基盤(固定トークン・セマンティック写像層・コンポーネント・検証手段)を再利用する。
- ガイドブックの 7 パレットは DS プリミティブ(`blue/light-blue/cyan/green/orange/red` + `solid-gray`)と同一系であり、当テンプレートは既にこれらを vendor 済み。整合は「カテゴリ配色をこれらのトークンで定義」して担保する。
- 機密性区分の既定は安全側(未設定時は最上位の注意表示、または明示のプレースホルダ)。既定サンプルは「機密性2情報」「（組織名）限定」とし、作成者が設定を変更する。
- ページ番号はデック(ショーケース/配布デック)で通し番号を表示。単体スライドファイルでは当該番号またはプレースホルダを表示。
- 取扱区分・機密性の区分名は日本語(政府の情報格付けの一般的表記)を既定とする。実際の格付け運用・法的判断は利用組織の規程に従う(テンプレートは表示枠を提供)。
- ダッシュボードの実データ・実チャートの値は作成者が差し込むプレースホルダ(作図エンジンは持たない、feature 001/002 の方針を踏襲)。

## Out of Scope

- 実データバインディング・BI ツール連携(ガイドブックの Power BI テンプレートに相当する動的機能)。
- 機密性区分の自動判定やアクセス制御(表示枠のみ提供、格付け判断は運用側)。
- アニメーション/画面遷移、編集可能 PowerPoint への実変換(feature 001 と同じくフェーズ2)。
