# Verification Contract: レイアウト描画の修正と構図の統一

このデックは外部 API を公開しない。契約は「配布物が満たすべき不変条件」と「本フィーチャーの受け入れチェック」で構成する。実装は本契約を破ってはならない。

## C1: 検証パイプライン(単一の正)

`npm run verify` が全通過すること。内訳と本フィーチャーでの意味:

| ステップ | 保証する不変条件 | 関連 |
|---|---|---|
| `lint:tokens` | 生の色/spacing px の非導入(`calc(var(--font-size-64)*N)` は通過) | FR-009 |
| `check:crossrefs` / `check:coverage` | 出典・実践の相互参照が不整合を生まない | 回帰防止 |
| `check:slides` | `index.html` から `slides/*.html` が再生成整合(手編集していない) | G3 |
| `test:visual` | ネイティブ寸法の視覚回帰(構図変更が意図どおり・見切れなし) | FR-004, SC-003 |
| `test:a11y` | axe-core でコントラスト等の違反0(濃地フレーム含む) | FR-003, SC-002 |
| `test:print` | 印刷レイアウトの1スライド=1ページ | FR-011 |
| `build:pdf` → `check:pdf` | PDF ページ数 == ライブ `.slide` 数(動的) | FR-010, SC-006 |

## C2: 受け入れチェック(スライド別・目視/PDF)

`npm run build:pdf` 後、`dist/deck.pdf` を Read ツールの `pages` で確認する。

| チェック | 期待 | 出典シナリオ |
|---|---|---|
| AC-1 付録D 凡例 | 企画=青 / 開発=緑 / 運用=橙 が系列と一致 | US1-1 |
| AC-2 全面画像フレーム | 右上「(組織名)限定」・左下 ©・右下ページ番号が判読可能 | US1-2 |
| AC-3 付録E/D 系列色 | 凡例・系列・データ点の対応が全一致 | US1-3 |
| AC-4 短文スライド | ビッグナンバー/まとめ/プロセス/タイムライン/図表/付録F で本文が見出し直下から開始 | US2-1 |
| AC-5 中央寄せ意図 | 章扉/引用/ビッグナンバーの中央寄り構図が保持 | US2-2 |
| AC-6 ビッグナンバー | 「16」が同一スライド内で最大要素・内訳と近接 | US2-3, SC-004 |
| AC-7(任意) 付録F/H・CTA・表紙 | 疎スライドの体裁整合・CTA 折り返し自然・表紙構図 | US3 |

## C3: 非回帰(禁止事項)

- `@media print` ルールおよび `tokens/vendor/**` を変更しない。
- スライドの追加・削除を行わない(既存25枚)。
- 発表モード(present)を既定化しない(既定 review を維持)。
- 中立凡例スウォッチ(`--muted`/`--surface`)の色を変えない。

## 合格条件

C1 全通過 かつ C2 の AC-1〜AC-6 充足(AC-7 は任意)。C3 に抵触しないこと。
