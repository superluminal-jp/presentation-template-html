# Contract: Claude Code 評価ルーブリック

Claude Code が生成 PDF（またはブラウザ表示）を視覚評価する際の判定基準とレポート様式を規定する契約。基準は本デックが既に掲げる設計原則（`docs/practices.md`、スライド注釈 `data-practice`、DS トークン）に依拠し、新たな基準体系は作らない。

## 評価対象範囲

- **含む**: 描画破綻、設計意図適合（視覚レベル）。
- **含まない**: 文言・メッセージ内容そのものの良し悪し（範囲外）。

## A. 描画破綻チェック（render）

各スライドについて以下を確認し、1 つでも該当すれば `render = FAIL`：

1. **見切れ**: 文字・図版・タイル等がキャンバス（16:9）外へはみ出す／端で切れている。
2. **ページ分裂**: 1 スライドが複数ページに分割されている（PDF ページ数=スライド枚数の逸脱）。
3. **フレーム欠落**: 共通フレーム（取扱区分・著作権・ページ番号）が表示されていない。
4. **要素の重なり・崩れ**: 要素同士の不正な重なり、グリッド崩れ、極端な余白破綻。
5. **文字化け・欠字**: フォント未適用・豆腐（□）・欠落。
6. **注釈混入**: 既定非表示の注釈レイヤーが出力に現れている。

判定不能（画像が不鮮明・低解像度で確認できない）は `render = UNEVALUABLE`。合格扱いにしない。

## B. 設計意図適合チェック（design）

各スライドについて、視覚レベルで以下への適合を確認し、明確な逸脱があれば `design = FAIL`：

1. **DS 準拠配色・トークン**: 配色がデックの基調（DS キーカラー等）から逸脱していない。ブランド外の色が現れていない。
2. **アクションタイトル**: 見出しが主張として機能している（体言止めの飾りでなく要点を述べている）ことが視覚上妨げられていない（見出しの可読・階層が保たれている）。
3. **視覚階層**: 見出し＞本文＞キャプションの階層が保たれ、単一焦点（アクセント多用でない）。
4. **1スライド1メッセージ**: 情報過多で要点が埋もれていない（例: ダッシュボードのタイルが 6 件以内で整列）。
5. **可読性・コントラスト**: 文字と背景のコントラストが十分で読める（重ね文字はスクリム等で確保）。

判定不能は `design = UNEVALUABLE`。

## C. レポート様式（出力契約）

スライド単位の表を出力する（人可読・機械可読の双方に扱える）：

| slide | render | design | findings |
|---|---|---|---|
| `<ページ番号-レイアウト>` | PASS/FAIL/UNEVALUABLE | PASS/FAIL/UNEVALUABLE | FAIL/UNEVALUABLE 時は具体症状と該当箇所（必須）。PASS は `—` |

末尾に集計と総合判定を付す：

- `total` / `renderFail` / `designFail` / `uneval`
- `verdict`: すべての `render` と `design` が `PASS` のときのみ「合格」。`UNEVALUABLE` が 1 件でもあれば合格にしない（FR-013）。

## D. 手続き（Claude Code の実行順序）

1. `npm run build:pdf` で `dist/deck.pdf` を生成。
2. `npm run check:pdf` で決定的下地チェック（枚数一致・生成成功）。FAIL ならそこで是正。
3. `dist/deck.pdf` を Read ツールの `pages` で全ページ読み込み、A/B を各スライドに適用。
4. （可能なら）ブラウザで `index.html` を開き、発表モード `P`／注釈トグル `A` を観察して所見補強。
5. C の様式でレポートを出力。FAIL があれば該当スライドと症状を明示。
